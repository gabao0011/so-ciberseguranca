Segurança Web e mobile

Como quem estuda DEV, você vai construir sites e apps. Vale conhecer os ataques mais comuns e como evitá-los no código.

OWASP Top 10 (resumido)

A OWASP mantém uma lista dos maiores riscos web. Os 3 clássicos que toda escola deveria ensinar:

SQL Injection

Atacante envia código SQL dentro de um campo. Se o sistema concatena strings, o banco executa comandos indesejados.

// INSEGURO
query = "SELECT * FROM users WHERE email = '" + email + "'";
// Se email = ' OR '1'='1 → retorna todos

// SEGURO (prepared statements)
query = "SELECT * FROM users WHERE email = ?";
stmt.bind(email);

XSS — Cross-Site Scripting

Atacante injeta JavaScript num site para roubar cookies, redirecionar ou enganar o usuário.

  • Sanitize input antes de mostrar (< vira &lt;).
  • Use Content Security Policy (CSP).
  • Cookies HttpOnly + Secure.

Broken Authentication

Sistemas que armazenam senha em texto, não bloqueiam após tentativas, não invalidam sessão...

Outros riscos

  • CSRF: forçar você a executar ação autenticada sem querer.
  • IDOR: trocar o ID na URL (/pedido/123) e ver o pedido de outro usuário.
  • Uploads perigosos: aceitar arquivos .php em pastas executáveis.
  • Dependências vulneráveis: atualize seu npm / composer.

Segurança mobile

  • Instale apps apenas da loja oficial.
  • Revise as permissões (por que um app de lanterna precisa de contatos?).
  • Ative o bloqueio de tela e a criptografia do dispositivo.
  • Cuidado com redes Wi-Fi desconhecidas.
  • Desative Bluetooth/NFC quando não usar.
Regra #1 para devs: Nunca confie em input do usuário. Sempre valide, sanitize e use prepared statements.
