Virtualização e Nuvem

Até aqui falamos de "um" sistema rodando em "um" computador. Mas hoje é super comum que vários S.O. rodem ao mesmo tempo na mesma máquina — é a base da computação em nuvem.

O que é virtualização?

Virtualização é criar uma máquina virtual (VM) — um computador "de mentirinha" dentro de outro. Ela tem CPU, RAM e disco simulados, e pode rodar qualquer S.O.

Quem cuida disso é o hypervisor (ex.: VMware, VirtualBox, Hyper-V, KVM).

Tipos de hypervisor

  • Tipo 1 (bare metal): roda direto no hardware. Usado em datacenters (VMware ESXi, Hyper-V, Xen, KVM).
  • Tipo 2 (hosted): roda como programa dentro de um S.O. comum (VirtualBox, VMware Workstation).

Por que usar VMs?

  • Isolamento: um problema em uma VM não derruba as outras.
  • Aproveitamento: um servidor potente pode hospedar dezenas de VMs.
  • Teste: você pode rodar Linux dentro do Windows sem reinstalar.
  • Segurança: ambiente sandbox para analisar malware.

Contêineres (Docker)

VMs simulam o hardware inteiro (pesado). Contêineres compartilham o kernel do hospedeiro e isolam só o necessário. São muito mais leves e iniciam em segundos.

Nuvem (Cloud)

A computação em nuvem é "alugar" poder computacional em datacenters via internet. Provedores famosos:

  • AWS (Amazon)
  • Azure (Microsoft)
  • Google Cloud Platform (GCP)
  • Oracle Cloud, IBM Cloud, etc.

Modelos de serviço:

  • IaaS (Infraestrutura como Serviço) — você aluga a VM.
  • PaaS (Plataforma) — você só envia o código; a plataforma roda.
  • SaaS (Software) — você só usa o aplicativo (Gmail, Netflix).
