Entrada e Saída (E/S)

Teclado, mouse, disco, rede, impressora, câmera… cada um desses dispositivos é tratado pelo S.O. com a ajuda de um driver e de um mecanismo chamado interrupção.

Categorias de dispositivos

  • Entrada: teclado, mouse, microfone, câmera, scanner.
  • Saída: monitor, impressora, alto-falante.
  • Entrada e saída: disco, pendrive, placa de rede.

Drivers

Um driver é um pequeno programa (normalmente do fabricante) que o kernel usa para conversar com um hardware específico. Sem driver, o sistema não sabe usar aquele dispositivo.

Comunicação: polling x interrupção

MétodoComo funcionaProblema
PollingCPU pergunta o tempo todo: "Chegou? Chegou? Chegou?"Gasta CPU à toa.
InterrupçãoO hardware avisa a CPU quando tem algo.Mais eficiente — usado hoje.

DMA — Direct Memory Access

Para não sobrecarregar a CPU copiando grandes volumes de dados, existe o DMA: o dispositivo escreve direto na RAM e só avisa a CPU quando termina.

Buffering, caching e spooling

  • Buffering: guarda um pedaço na RAM antes de entregar/enviar (ex.: vídeo carregando).
  • Caching: guarda uma cópia do que foi lido recentemente, pra não ter que ir no disco de novo.
  • Spooling: fila de arquivos esperando o dispositivo (ex.: fila de impressão).

Por que o pendrive "piscou"?

O sistema escreve em cache e só de vez em quando manda tudo para o pendrive. Se você arranca na hora errada, os dados podem se perder. Por isso existe o "remover com segurança" — ele força o flush do cache.
