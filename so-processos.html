Processos e Threads

Quando você clica no ícone do Chrome, o S.O. cria um processo. Ele é uma "cópia do programa rodando", com sua própria memória, variáveis e estado. Dentro dele, várias threads podem executar em paralelo.

O que é um processo?

Um processo é um programa em execução. Ele tem:

  • Código (as instruções).
  • Dados (variáveis globais).
  • Pilha (stack) e heap (áreas de memória).
  • PID — identificador único (número).
  • PCB (Process Control Block) — estrutura do kernel com estado, registradores, prioridade, etc.
Programa ≠ Processo. Um programa é o arquivo no disco (ex.: chrome.exe). Um processo é uma execução dele na memória. Você pode ter vários processos do mesmo programa rodando (várias abas, várias janelas).

Estados de um processo

EstadoO que significa
NovoAcabou de ser criado, sendo inicializado.
ProntoEsperando a CPU liberar. Já tem tudo pra rodar.
ExecutandoEstá na CPU agora, executando instruções.
Bloqueado / EsperandoParou para esperar algo (disco, rede, teclado).
TerminadoFinalizou sua execução.

Transições típicas

  • Pronto → Executando: escalonador escolheu este processo.
  • Executando → Bloqueado: pediu algo (ler arquivo) e está esperando.
  • Bloqueado → Pronto: o que esperava chegou (ex.: dado do disco).
  • Executando → Pronto: S.O. cortou o tempo dele (preempção).

Threads (linhas de execução)

Uma thread é um "fluxo de execução" dentro de um processo. Um processo pode ter várias threads rodando em paralelo, compartilhando memória.

  • Vantagens: mais rápido criar uma thread do que um processo; compartilham dados facilmente.
  • Cuidado: como compartilham memória, podem se atrapalhar (condição de corrida).
Exemplo: Um navegador moderno costuma ter um processo por aba (isolamento + segurança), e cada aba tem várias threads — uma para render, uma para JavaScript, uma para rede, etc.

Concorrência x Paralelismo

  • Concorrência: parece que várias coisas acontecem ao mesmo tempo (o S.O. alterna rápido).
  • Paralelismo: várias coisas acontecem de verdade ao mesmo tempo (precisa de mais de um núcleo).

Comandos úteis

  • Windows: Gerenciador de Tarefas (Ctrl+Shift+Esc).
  • Linux: ps aux, top, htop.
  • macOS: Monitor de Atividade.
