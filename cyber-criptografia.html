Criptografia

Criptografia é a matemática que embaralha dados para que só quem tem a chave consiga ler. Está por trás do HTTPS, do WhatsApp, do Pix e até do login do Google.

Termos-chave

  • Texto claro: a mensagem original.
  • Texto cifrado: a versão embaralhada.
  • Chave: segredo usado para cifrar e decifrar.
  • Algoritmo: a "receita" matemática (AES, RSA, etc.).

Simétrica x Assimétrica

SimétricaAssimétrica
ChavesUma só — mesma pra cifrar e decifrarDuas — pública e privada
VelocidadeRápidaMais lenta

ExemplosAES, ChaCha20RSA, ECC

Na prática, os sistemas misturam as duas: usam assimétrica pra trocar uma chave simétrica segura, e simétrica pra cifrar os dados reais (por ser mais rápida). É assim que o HTTPS funciona.

Funções hash

Hash é um "resumo" do conteúdo. Transforma qualquer entrada em uma string fixa. É irreversível: não dá pra voltar do hash pro original.

  • Usado para armazenar senhas (nunca guarde senha em texto claro).
  • Verificar integridade de arquivos (baixou um ISO? Compare o hash).
  • Algoritmos: SHA-256, SHA-3. (MD5 e SHA-1 são antigos e quebrados).

Assinatura digital

Combina hash + criptografia assimétrica pra provar quem assinou e que o conteúdo não foi alterado. É como uma assinatura de papel, só que à prova de falsificação.

HTTPS em 3 passos

  1. Seu navegador recebe o certificado do site (com chave pública).
  2. Gera uma chave simétrica temporária e envia cifrada com a chave pública.
  3. A partir daí, os dois conversam usando criptografia simétrica rápida e segura.

Minigame — Cifra de César

Deslize cada letra N posições no alfabeto. Fácil de entender, péssima pra segurança — mas ótima para aprender o conceito.
